# H4I Website Project Context

## Overview
A minimal website for Hack4Impact (H4I), built with Next.js 15, React 19, Tailwind CSS 4, and BaseUI React 1.1.0. The project uses TypeScript and is managed with pnpm.

## Tech Stack
- Next.js 15.3.0 (App Router, Turbopack for dev)
- React 19.0.0
- Tailwind CSS 4 (configured via CSS @theme, not tailwind.config.js)
- BaseUI React 1.1.0 (@base-ui/react)
- Contentful CMS 11.5.5 (for content management)
- TypeScript 5

## Fonts
Fonts are self-hosted (not using next/font).

### Inclusive Sans
- Source: GitHub repo (LivKing/Inclusive-Sans) - NOT Google Fonts (Google version lacks ss01)
- Files: /public/fonts/InclusiveSans.ttf, InclusiveSans-Italic.ttf
- Format: TTF (TrueType)
- Weight range: 300-700 (variable)
- Usage: `font-sans` class, default body font
- Stylistic set ss01 (double-story "a") enabled in @font-face declaration via font-feature-settings

### Newsreader
- Source: Google Fonts
- Files: /public/fonts/Newsreader.woff2, Newsreader-Italic.woff2
- Format: woff2
- Weight range: 200-800 (variable)
- Usage: `font-serif` class

Font definitions are in globals.css using @font-face declarations. Tailwind theme variables:
- --font-sans: "Inclusive Sans", system-ui, sans-serif
- --font-serif: "Newsreader", Georgia, serif

## Components

### Button (src/components/ui/Button.tsx)
Wraps BaseUI's Button component with styling.

Props:
- variant: "primary" | "secondary" | "outline" (default: "primary")
- size: "sm" | "md" | "lg" (default: "md")
- All standard button props from BaseUI

Base styles:
- No border radius (square corners)
- Monospaced font (font-mono)
- Focus ring uses green-300

Variant styles:
- primary: bg-green-300, hover:bg-green-400, active:bg-green-500, black text
- secondary: bg-purple-600, hover:bg-purple-700, active:bg-purple-800, white text
- outline: border-green-300, text-green-300, hover:bg-green-50

Usage:
```tsx
import { Button } from "@/components/ui/Button";
<Button variant="primary" size="md">Click me</Button>
```

## Pages

### Home Page (src/app/page.tsx)
Current structure:
1. Intro Section
   - White-to-gray gradient background (from-white to-gray-200)
   - Gradient stops at bottom-1/3, allowing image to hang below
   - Bottom padding pb-32 ensures image hangs off gradient on mobile
   - Top padding pt-24 for gap between navbar and content
   - Stacked title (flexbox column, no gap):
     - "Code & community" (Newsreader/serif, text-4xl md:text-6xl) on top
     - "for the common good" (Inclusive Sans/sans, text-4xl md:text-6xl) below
     - Both lines same font size, tight spacing to read as one title
   - Subtitle paragraph (text-base md:text-lg) about H4I mission
   - "Partner with us!" button (primary variant)
   - Full-width image (gt.jpg) that hangs below the gradient

### About Page (src/app/about/page.tsx)
Current structure:
1. Intro Section
   - Two-column grid layout (stacked on mobile), min-h-[70vh]
   - Left column: placeholder gradient (blue-100 to green-300, diagonal)
   - Right column: brand white background (#FCF9F2), vertically centered, left-aligned content
     - Title: "Creating software to support those supporting their communities" (sans-serif, text-3xl md:text-4xl)
     - Button: "Meet the Team" (primary variant)

2. Values Section
   - Title: "Our Values" (centered, text-3xl)
   - Three-column grid of value cards (stacked on mobile)
   - Each card (centered content):
     - Placeholder icon (gray circle, w-16 h-16)
     - Title: Sustainable / Ethical / Accessible (text-xl)
     - Description paragraph

```css
  border: 1px solid #ccc;
}
```
Remove these for production.
